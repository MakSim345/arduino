Энергонезависимая память EEPROM в Arduino

EEPROM (Electrically Erasable Programmable Read-Only Memory) — это тип памяти, который сохраняет данные при отключении питания микроконтроллера [cite: 1]. Используется для хранения настроек, калибровочных коэффициентов, счетчиков и т.д.

Основные характеристики и ограничения:

    Объем памяти: Зависит от чипа (например, у ATmega328 / Arduino Nano/Uno — всего 1 КБ).
    Ресурс: Ограниченное количество циклов записи (около 100 000 раз). Количество циклов чтения не ограничено.
    Скорость: Запись происходит относительно медленно (около 3.3 мс на один байт).

Схема / Структура памяти EEPROM (1 КБ)

Память можно представить как линейный массив ячеек, где каждая ячейка имеет свой адрес и вмещает ровно 1 байт данных (число от 0 до 255).

 Адрес ячейки:    [ 0 ]   [ 1 ]   [ 2 ]   [ 3 ]  ...  [ 1023 ]
                 ┌───────┬───────┬───────┬───────┬───┬────────┐
 Содержимое:     │  0x4A │  0xFF │  0x00 │  0x12 │   │  0x00  │  <- Данные (8 бит)
                 └───────┴───────┴───────┴───────┴───┴────────┘

Базовый инструментарий (Библиотека <EEPROM.h>)

Для работы с памятью в Arduino IDE стандартно используется встроенная библиотека

1. Классические функции (чтение/запись по 1 байту):

    EEPROM.write(address, value); — записывает 1 байт по указанному адресу.

    EEPROM.read(address); — считывает 1 байт по адресу.

    Важно: Если постоянно использовать write() в цикле loop(), можно быстро "убить" ячейку памяти.

2. Оптимизация ресурса памяти:

    EEPROM.update(address, value); — Рекомендуется вместо write!
    ->Проверяет текущее значение ячейки. Запись происходит только если новое значение отличается от уже записанного. Это экономит ресурс циклов перезаписи.

3. Работа со сложными данными (многобайтовые типы: float, int, структуры):

Поскольку read/write работают только с 1 байтом, для хранения больших переменных (например, int занимает 2 байта, float — 4 байта) используют универсальные методы:

    EEPROM.put(address, data); — записывает объект любого типа (автоматически вычисляет размер и использует update).

    EEPROM.get(address, data); — считывает объект любого типа из памяти в переменную.

Пример кода
--------------------------------------------------------------
#include <EEPROM.h>

struct MySettings
{
    float calibrationK;
    int modeID;
    char deviceName[10];
};

void setup()
{
    Serial.begin(9600);

    // WRITE
    int ledBrightness = 155;
    // Use update to save cell 0 resoiurce
    EEPROM.update(0, ledBrightness);

    MySettings customConfig = {3.14, 2, "Sensor_A"};
    // Write the whole structure starting from address 10
    EEPROM.put(10, customConfig);
}

void loop()
{
    // Better to keep empty to keep cell resources
    // READ:
    int readValue = EEPROM.read(0);

    MySettings savedConfig;
    EEPROM.get(10, savedConfig);

    // Serial printing:
    Serial.println(readValue);
    Serial.println(savedConfig.calibrationK);
}
--------------------------------------------------------------

Резюме: Всегда используй "update()" вместо "write()" для одиночных байт, а для тяжелых переменных и структур бери связку "put()/ .get()".
